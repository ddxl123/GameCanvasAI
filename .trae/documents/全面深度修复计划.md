# 游戏设计项目 全面深度修复计划

## 概述

本计划覆盖前序分析发现的 **全部 60+ 个问题**（P0 高危 15 个 + P1 中危 30+ 个 + P2 低危 15+ 个），按 8 个工作流组织，引入必要新依赖（zod / dompurify / gpt-tokenizer），最终通过构建验证 + 关键功能手动验证。

## 当前状态分析

项目是 React 18 + TypeScript + Vite + Zustand + Dexie + ReactFlow 的游戏设计工具。核心问题集中在：

1. **数据完整性**：导出/快照/删除仅覆盖 9/20 张表，11 张表数据静默丢失或成孤儿
2. **安全**：前端直连 AI 明文 Key + MarkdownRenderer/applyGddAction 多处 XSS + prompt 注入
3. **撤销系统**：自定义 history 的并发竞态与跨作用域污染未处理
4. **画布交互**：customLinks 悬空边、选中状态多源、memo 失效、无快捷键/错误边界
5. **工程**：TypeScript 7 RC 导致 lint 崩溃、包体 2.9MB 无分割、strict 关闭

关键文件：
- DB schema：`src/db/index.ts`（20 张表，v5）
- 导出：`src/lib/projectExport.ts`（仅 9 张表）
- 快照：`src/stores/snapshotStore.ts`（仅 9 张表，restore 不刷新 store）
- 撤销：`src/stores/historyStore.ts`（并发竞态 L70-98）
- AI 客户端：`src/lib/aiClient.ts`（无超时）、`src/services/aiService.ts`（env Key 高危，重复实现）
- XSS：`src/features/ai/MarkdownRenderer.tsx`（链接无协议白名单 L33-36）、`src/lib/aiActions.ts`（applyGddAction L1148 原样 HTML）
- 删除：`src/stores/projectStore.ts`（deleteProject 仅 9 表 L56-68）

## 假设与决策

- **新依赖**：`zod`（schema 校验）、`dompurify`（HTML 净化）、`gpt-tokenizer`（token 计数）。均为成熟纯前端库，体积可控。
- **不引入后端**：AI Key 仍前端持有（受限于纯前端架构），但加密存储 + 文档警告 + 废弃 env 路径。后端代理列为未来工作。
- **TypeScript 版本**：从 `7.0.1-rc` 降到稳定版 `^5.6.0`，修复 lint 崩溃。
- **strict 开启**：逐步开启 `strict: true`，修复暴露的类型问题。
- **DB 版本**：新增 v6 做数据迁移（补 groupId 等字段），不破坏现有数据。
- **验证**：每个工作流完成后跑 `npm run check && npm run build`；全部完成后对导出/快照/撤销/AI/XSS 做手动验证。

### 4.1 undo/redo 并发互斥锁（P0）

**文件**：`src/stores/historyStore.ts`

**What**：引入 isRunning 锁，防止并发 undo/redo。

**How**：
```ts
interface HistoryState {
  // ...existing
  isRunning: boolean;
}
undo: async () => {
  if (get().isRunning || get().past.length === 0) return;
  set({ isRunning: true });
  try {
    const cmd = get().past[get().past.length - 1];
    await cmd.undo();
    set((s) => ({ past: s.past.slice(0,-1), future: [...s.future, cmd] }));
  } catch (e) { console.error("撤销失败:", e); }
  finally { set({ isRunning: false }); }
}
```
